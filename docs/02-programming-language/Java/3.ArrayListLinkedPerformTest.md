# ArrayList vs LinkedList Performance Test
```java
public class ListPerformanceTest {
    
    private static final int N = 100_000; // 100,000 operations
    
    public static void main(String[] args) {
        // 1. Tail insertion test
        testTailAdd();
        
        // 2. Head insertion test (ArrayList's death trap)
        testHeadAdd();
        
        // 3. Random access test (LinkedList's death trap)
        testRandomAccess();
    }
    
    static void testTailAdd() {
        List<Integer> arr = new ArrayList<>();
        List<Integer> link = new LinkedList<>();
        
        long t1 = System.nanoTime();
        for (int i = 0; i < N; i++) arr.add(i);
        long arrTime = System.nanoTime() - t1;
        
        long t2 = System.nanoTime();
        for (int i = 0; i < N; i++) link.add(i);
        long linkTime = System.nanoTime() - t2;
        
        System.out.println("=== testTailAdd " + N + " ===");
        System.out.println("ArrayList:  " + arrTime / 1_000_000 + " ms");
        System.out.println("LinkedList: " + linkTime / 1_000_000 + " ms");
        System.out.println();
    }
    
    static void testHeadAdd() {
        List<Integer> arr = new ArrayList<>();
        List<Integer> link = new LinkedList<>();
        
        long t1 = System.nanoTime();
        for (int i = 0; i < N; i++) arr.add(0, i); // Insert at head
        long arrTime = System.nanoTime() - t1;
        
        long t2 = System.nanoTime();
        for (int i = 0; i < N; i++) link.add(0, i); // Insert at head
        long linkTime = System.nanoTime() - t2;
        
        System.out.println("=== testHeadAdd " + N + " ===");
        System.out.println("ArrayList:  " + arrTime / 1_000_000 + " ms  (Slow! Because all elements need to be shifted)");
        System.out.println("LinkedList: " + linkTime / 1_000_000 + " ms  (Fast! Because only pointers are modified)");
        System.out.println();
    }
    
    static void testRandomAccess() {
        List<Integer> arr = new ArrayList<>();
        List<Integer> link = new LinkedList<>();
        
        // Fill with data first
        for (int i = 0; i < N; i++) {
            arr.add(i);
            link.add(i);
        }
        
        long t1 = System.nanoTime();
        long sumArr = 0;
        for (int i = 0; i < N; i++) sumArr += arr.get(i);
        long arrTime = System.nanoTime() - t1;
        
        long t2 = System.nanoTime();
        long sumLink = 0;
        for (int i = 0; i < N; i++) sumLink += link.get(i); // LinkedList.get(i) is O(n)!
        long linkTime = System.nanoTime() - t2;
        
        System.out.println("=== testRandomAccess " + N + " ===");
        System.out.println("ArrayList:  " + arrTime / 1_000_000 + " ms  (Fast! Direct array indexing)");
        System.out.println("LinkedList: " + linkTime / 1_000_000 + " ms  (Slow! Every get traverses from the beginning)");
        System.out.println();
        
        // Prevent compiler from optimizing away sum
        System.out.println("Checksum: " + (sumArr == sumLink));
    }
}
```

Output:
```
=== testTailAdd 100000 ===
ArrayList:  4 ms
LinkedList: 2 ms

=== testHeadAdd 100000 ===
ArrayList:  272 ms  (Slow! Because all elements need to be shifted)
LinkedList: 3 ms  (Fast! Because only pointers are modified)

=== testRandomAccess 100000 ===
ArrayList:  1 ms  (Fast! Direct array indexing)
LinkedList: 5122 ms  (Slow! Every get traverses from the beginning)

Checksum: true
```

| Operation     | ArrayList | LinkedList | Performance Gap             |
| ------------- | --------- | ---------- | --------------------------- |
| `add` at tail | 4 ms      | 2 ms       | Roughly the same            |
| `add` at head | 272 ms    | 3 ms       | ArrayList is ~90× slower    |
| Random `get`  | 1 ms      | 5406 ms    | LinkedList is ~5400× slower |

This is the truth of the underlying data structures:

- ArrayList = dynamic array. <br />
Head insertion requires shifting all subsequent elements. Random access calculates the address directly.

- LinkedList = doubly linked list. <br />
Head insertion only changes two pointers. Random access must traverse from the beginning to the i-th node.

When decide which to use in the future, let below guide the choice:

- Frequent insertion/deletion at the front? → Use LinkedList (or ArrayDeque).
- Frequent random access? → Use ArrayList.
- Only appending at the tail? → Both work, but ArrayList is more memory-efficient.
